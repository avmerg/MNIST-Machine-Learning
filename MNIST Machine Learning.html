Prompt:

Explain either the mathematics or the intuition behind the kernel trick used in SVMs and how it allows to express a non-linear classifier through a linear classifier in a higher-dimensional space.

Answer:

The kernel trick is used when our data is in 'n' dimensional space, which is not separable into linearly boundaries by a Support Vector Classifier. As 'real-life' data is often not linearly separable, the use of kernel functions is key for defining boundaries. Optimal decision boundaries are created from the maximum possible distance between points in different classes in the linear case, but when data is in a hyperplane with 'n' dimensions, the constraints of the decision boundary changes.

Instead, if the decision boundary in a hyperplane that separates classes has different weights in a vector v, it is necessary to maximize the distance from this vector to the closest points (the support vectors). The kernel trick avoids a step of mapping linear functions to higher dimensional space because these functions can be expressed instead as the dot product of data in the higher space instead of operating on the data itself (Kandan 2017). This mapping is one of the implications of Mercer's theorem, which states that the inner products on a finite dimensional space can be characterized by linear operators (Thickstun 2017). The use of these inner products (i.e. the "kernel trick") considerably increases computational efficiency of separating data into n-dimensional space and allows the kernel functions of choice to designate more appropriate decision boundaries around the data.

The most common types of kernels used are linear, polynomial, radial basis function (RBF), and sigmoid kernel. The different kinds of kernel are specified as follows (d, γ, r are kernel parameters):

Linear Kernel:

$$k(X,Y) = X^TY + c$$

Polynomial kernel: This function is often used in image processing (where γ>0 and d is the polynomial degree of the data)

$$K(X,Y)= (x^T y + c)^d$$

Radial Basis function (RBF) Kernel: Often this kind of kernel function is used in situations where not much context about the data is known (where γ>0):

$$K(X,Y)=(−γ⋅‖X−Y‖^2)$$

Sigmoid Kernel: This kernel function operates similarly to a sigmoid function in a logistic regression.

$$K(X,Y)=tanh(γ⋅X^TY+r)$$

Prompt:

By applying the chain law of derivatives to the objective function of a neural network, derive the back propagation algorithm.

Answer:

At the core of the back propagation algorithm, it takes the cost function, C and takes the partial derivative with respect to any weight (w) or bias (b) in the network (∂C/∂w). The back propogation algorithm in practice tells us how quickly the cost (accuracy) changes with changes to the weights and biases in the neural network. Back propagation calculates an error at the end of the calculation and then adjusts each weight in a network in proportion to that weight's contribution to the overall error. In adjusting or penalizing these weights iteratively, the algorithm is able to adjust weights so that the neural network creates good predictions.

The backpropagation algorithm can be thought of a way to apply the chain rule to find the derivatives of the cost function with respect to any variable (bias or weight)in a nested equation.

To illustrate this principle, in the case of a forward propagation function, if A1, A2, and A3 are all activation functions at different layers of a neural network, a function could look like this: $$f(x)=A1(A2(A3(x)))$$

If the derivative of f(x) is taken with respect to x, using the chain rule, we can calculate $$f′(x)=f′(A1)⋅A1′(A2)⋅A2′(A3)⋅A3′(x)$$

To take the derivative with respect to $A2$, you put $A2$ as a placeholder in the function, pretending that $A2(A3(x))$ is a constant, and then find the derivative with respect to $A2$ normally. This can help us investigate the influence a variable has on the total output, in this case, the cost function of a neural network for backpropagation.

In order to figure out how much weights are contributing to error and by how much, several functions need to be defined.

Defining Weighted Input $$ L(X,W) = XW $$ $$ L'(X) = W $$ $$ L'(W) = X $$ ReLu Activation Function $$ R(L) = max(0,L)$$ $$ R'(L) = 0; L<0$$ $$ R'(L) = 1; L>0$$ Cost Function $$ C (\hat{y})= (1/2)(\hat{y} - y)^2$$ $$ C'(\hat{y}) = (\hat{y} - y) $$

If a network had a single neuron, total cost could then be described as: $$ Cost=C(R(L(X,W))) $$

Which, if you apply the chain rule to it, becomes: $$ C′(W)=C′(R)⋅R′(L)⋅L′(W)=(ŷ −y)⋅R′(L)⋅X $$

Chain of events in the neural network:

The input layer to the hidden layer: $$ L_{h} = X * W_{h} $$

The hidden layer activation function: $$ H = R(L_{h}) $$

The hidden layer to the last layer before the output: $$ L_{0} = H * W_{0} $$

The last input to the output function: $$ O = R(L_{0}) $$

First, we'll consider a neural network with one hidden layer. In an example where $W_{0}$ is the last weight before the output layer, we can see the derivative with respect to this last layer by the following:

$$C′(W_{0})=C′(ŷ )⋅ŷ ′(L_{0})⋅L′_{0}(W_{0})=(ŷ −y)⋅R′(L_{0})⋅H $$

Then we can find out the derivatives even further back in our function by applying this chain rule recursively (where we get to a function with the $W_{h}$'s term if H signifies the number of hidden layers and ${h}$ is the index).

$$C′(W_{h})=C′(ŷ )⋅O′(L_{0})⋅L′_{0}(H)⋅H′(L_{h})⋅L′_{h}(W_{h})=(ŷ −y)⋅R′(L_{0})⋅W_{0}⋅R′(L_{h})⋅X$$

Similarly, if we wanted to find out the cost relative to last output layer $L_{o}$ (i.e. how the last inputs effected the cost of the network), we would calculate:

$$C′(L_{0})=(ŷ −y)⋅R′(L_{0})$$

We would then calculate the cost of the inputs relative to any hidden layer, $H$ with respect to the hidden layer input, $L_{h}$.

$$C′(L_{h})=(ŷ −y)⋅R′(L_{0})⋅W_{0}⋅R′(L_{h})$$

This formula is commonly simplified to:

$$ E_{h}=E_{0}⋅W_{0}⋅R′(L_{h}) $$

This formula shows us that we are calculating a layer's error and then passing along the weight's errors back across the previous layer, updating the weights every time while using the derivative of cost function.

Finally, simplifying even further, we can take the previous function and substitute in the new $E_{0}$ term: $$C′(W_{0})=(ŷ −y)⋅R′(L_{0})⋅H $$

Turns into:

$$C′(W_{o})=E_{0}⋅H$$

Thus, in order to find out the derivative of cost with respect to any weight, we can multiply the current layer's error and the current layer's input.

$$C′(w)=CurrentLayersError⋅CurrentLayersInput$$

We see these principles implemented in the form of matrix multiplication and NumPy operations in the next section.

Max pooling layers use the greatest value of clusters of neurons at the prior layer. Just like other kinds of neural networks, the fully-connected layer does the primary executive "work" of the neural network, with access to the all the activations in the layer before (Karpathy 2018). A ReLu activation function was used, and a softmax layer was also used, as it is a categorical classification task. We were able to achieve 98.8% accuracy over running the convolutional neural network over five epochs. When validating against the test set, we achieved 98.71% test accuracy and .039 test loss.

x_train = x_train.reshape(x_train.shape[0], x_train.shape[1], x_train.shape[2], 1)
x_test = x_test.reshape(x_test.shape[0], x_test.shape[1], x_test.shape[2], 1)
y_train_h = pd.get_dummies(y_train)
model = tf.keras.Sequential()
model.add(tf.keras.layers.Conv2D(15, kernel_size=(3, 3),
                 activation='relu',
                 input_shape=(28,28,1)))
model.add(tf.keras.layers.MaxPooling2D(pool_size=(2, 2)))
model.add(tf.keras.layers.Conv2D(20, (3, 3), activation='relu'))
model.add(tf.keras.layers.MaxPooling2D(pool_size=(2, 2)))
model.add(tf.keras.layers.Flatten())
model.add(tf.keras.layers.Dense(120, activation='relu'))
model.add(tf.keras.layers.Dropout(0.2))
model.add(tf.keras.layers.Dense(84, activation='relu'))
model.add(tf.keras.layers.Dropout(0.2))
model.add(tf.keras.layers.Dense(10, activation='softmax'))
model.compile(optimizer=tf.train.AdamOptimizer(), 
          loss='categorical_crossentropy',
          metrics=['accuracy'])
model.fit(x_train, y_train_h,
          batch_size=128,
          epochs=5,
          verbose=1,
          validation_split = 0.2,
          shuffle=True)
